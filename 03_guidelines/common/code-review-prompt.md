# Code Review Prompt

A prompt template for conducting guideline-based reviews on code changes.

---

## Prompt

```
Please perform a code review on the following changes.

## Review Perspectives

Review against project guidelines in priority order:
1. Security (common/security.md)
2. Type Safety (common/code.md)
3. Error Handling (common/error-handling.md)
4. AI Coding Pitfalls (common/ai-coding.md) — phantom imports, inconsistent patterns, missing edge cases
5. Architecture (02_decision-criteria/architecture.md)
6. Naming (common/naming.md)
7. Performance (common/performance.md)
8. Abstraction (02_decision-criteria/abstraction.md)

## AI-Generated Code Specific Checks
If the code was generated by AI, also check:
- [ ] No phantom imports (libraries that don't exist in package.json)
- [ ] No outdated API usage (API matches the installed version)
- [ ] No unnecessary abstraction (justified by actual reuse, not hypothetical)
- [ ] Consistent with existing patterns in the same module

## Output Format

| File | Line | Perspective | Severity | Issue |
|------|------|-------------|----------|-------|
| path/to/file.ts | 25 | Security | ❌ Required | Missing auth() check |
| path/to/file.ts | 42 | Type Safety | ⚠️ Recommended | Return type should be explicit |

### Severity Definitions
- ❌ **Required**: Issues related to security or data integrity. Must be fixed before merging
- ⚠️ **Recommended**: Issues related to quality or maintainability. Should be fixed
- 💡 **Suggestion**: Suggestions for better approaches. Optional to address
```
