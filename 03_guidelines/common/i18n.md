# Internationalization (i18n) Guidelines

This document defines the design policy and implementation patterns for internationalization in Python applications (FastAPI / CLI).

---

## 1. Core Policy

### Adopted Libraries

- **gettext**: Python standard library for message translation (`.po` / `.mo` files)
- **Babel**: Locale-aware formatting (dates, numbers, currencies) and message extraction
- **FastAPI integration**: Locale detection from `Accept-Language` header, per-request translation context

### Default Language Detection

The default language should be determined dynamically based on the application type:

| Application Type | Detection Method | Fallback |
|------------------|-----------------|----------|
| Web application (FastAPI) | `Accept-Language` header or `?lang=` query parameter | en |
| CLI tool / Desktop app | OS locale (`LANG`, `LC_ALL`, or system API) | en |

- **Web application**: Use the `Accept-Language` header to detect the user's preferred language per request. Optionally accept a `?lang=` query parameter for explicit override. Store the preference in a cookie or user settings for subsequent visits.
- **CLI tool / Desktop application**: Read the OS locale environment variables (`LANG`, `LC_ALL` on Unix, `GetUserDefaultUILanguage` on Windows) to determine the default language at startup.

### Supported Languages (Initial)

| Code | Language | Priority |
|--------|------|--------|
| en | English | Fallback default |
| ja | Japanese | Required |
| zh | Chinese (Simplified) | Optional |
| ko | Korean | Optional |

## 4. i18n Setup and Helpers

### app/i18n/**init**.py

```python
import gettext as gettext_module
from contextvars import ContextVar
from pathlib import Path

from app.i18n.config import LOCALES_DIR, DEFAULT_LOCALE, SUPPORTED_LOCALES

# Context variable for per-request locale
_current_locale: ContextVar[str] = ContextVar("current_locale", default=DEFAULT_LOCALE)


def get_current_locale() -> str:
    return _current_locale.get()


def set_current_locale(locale: str) -> None:
    if locale not in SUPPORTED_LOCALES:
        locale = DEFAULT_LOCALE
    _current_locale.set(locale)


def get_translator(locale: str | None = None) -> gettext_module.GNUTranslations:
    """Get a translator for the given locale."""
    loc = locale or get_current_locale()
    try:
        return gettext_module.translation(
            "messages",
            localedir=str(LOCALES_DIR),
            languages=[loc],
        )
    except FileNotFoundError:
        return gettext_module.NullTranslations()


def gettext(message: str) -> str:
    """Translate a message using the current locale."""
    return get_translator().gettext(message)


def ngettext(singular: str, plural: str, n: int) -> str:
    """Translate a message with plural support."""
    return get_translator().ngettext(singular, plural, n)


# Convenience aliases
_ = gettext
```

## 5. Locale Detection Middleware

### app/api/middleware/locale.py

```python
from fastapi import Request
from starlette.middleware.base import BaseHTTPMiddleware

from app.i18n import set_current_locale
from app.i18n.config import DEFAULT_LOCALE, SUPPORTED_LOCALES


class LocaleMiddleware(BaseHTTPMiddleware):
    """Detect locale from Accept-Language header or query parameter."""

    async def dispatch(self, request: Request, call_next):
        locale = self._detect_locale(request)
        set_current_locale(locale)
        response = await call_next(request)
        response.headers["Content-Language"] = locale
        return response

    def _detect_locale(self, request: Request) -> str:
        # 1. Check query parameter (?lang=en)
        lang_param = request.query_params.get("lang")
        if lang_param and lang_param in SUPPORTED_LOCALES:
            return lang_param

        # 2. Check Accept-Language header
        accept_language = request.headers.get("accept-language", "")
        for part in accept_language.split(","):
            lang = part.split(";")[0].strip().split("-")[0]
            if lang in SUPPORTED_LOCALES:
                return lang

        return DEFAULT_LOCALE
```

### Register Middleware

```python
# app/main.py
from fastapi import FastAPI
from app.api.middleware.locale import LocaleMiddleware

app = FastAPI()
app.add_middleware(LocaleMiddleware)
```

## 7. Usage Patterns

### Usage in FastAPI Endpoints

```python
# app/api/routes/projects.py
from fastapi import APIRouter
from app.i18n import _

router = APIRouter()


@router.post("/api/projects")
async def create_project(data: ProjectCreate):
    try:
        project = await project_service.create(data)
        return {"id": project.id, "message": _("Create New Project")}
    except Exception:
        return {"error": _("A server error occurred")}
```

### Usage in Service Layer

```python
# app/features/project/service.py
from app.i18n import _, ngettext


def get_project_summary(count: int) -> str:
    return ngettext(
        "{count} project",
        "{count} projects",
        count,
    ).format(count=count)
```

### Variable Interpolation

```python
from app.i18n import _

# messages.po: msgid "{field} is required"
message = _("{field} is required").format(field="Email")
# → "Email is required" (en) / "Emailは必須です" (ja)
```

### Date & Number Formatting with Babel

```python
from babel.numbers import format_currency, format_decimal
from babel.dates import format_datetime, format_timedelta
from datetime import datetime, timedelta
from app.i18n import get_current_locale


def format_price(amount: float, currency: str = "USD") -> str:
    locale = get_current_locale()
    return format_currency(amount, currency, locale=locale)
    # → "$1,000.00" (en) / "＄1,000" (ja)


def format_date(dt: datetime) -> str:
    locale = get_current_locale()
    return format_datetime(dt, format="long", locale=locale)
    # → "January 15, 2024 at 2:30:00 PM UTC" (en) / "2024年1月15日 14:30:00 UTC" (ja)


def format_relative(delta: timedelta) -> str:
    locale = get_current_locale()
    return format_timedelta(delta, locale=locale)
    # → "3 days" (en) / "3日" (ja)
```

    return {"message": _("Loading...")}

## 9. Compilation and Extraction Workflow

### Extract Messages from Source Code

```bash
# Extract translatable strings from Python files
pybabel extract -F babel.cfg -o locales/messages.pot app/

# Initialize a new locale (first time only)
pybabel init -i locales/messages.pot -d locales -l en
pybabel init -i locales/messages.pot -d locales -l ja

# Update existing locale files with new strings
pybabel update -i locales/messages.pot -d locales

# Compile .po files to .mo (required for runtime)
pybabel compile -d locales
```

### Automate in CI

```yaml
# In GitHub Actions workflow
- run: uv run pybabel compile -d locales
- run: uv run pytest tests/i18n/
```

## 10. Message Management Best Practices

### Key Naming Rules

Use descriptive, unambiguous source strings. The `msgid` is the English source text (or a descriptive key):

```po
# ❌ Ambiguous
msgid "button1"
msgstr "Save"

# ✅ Clear: use the actual English text as the msgid
msgid "Save"
msgstr "保存"

# ✅ For context-dependent strings, use pgettext
# pgettext("button", "Save")  — distinguishes "Save" button from "Save" noun
```

### Translating Dynamic Content

```python
# ❌ Dynamically generating translation strings (loses extractability)
_(f"status.{status}")

# ✅ Use a mapping
STATUS_MESSAGES = {
    "pending": _("Pending"),
    "completed": _("Completed"),
    "failed": _("Failed"),
}

def get_status_message(status: str) -> str:
    return STATUS_MESSAGES.get(status, status)
```

## 11. Testing

### Translation Key Existence Check

```python
# tests/i18n/test_messages.py

import polib
import pytest
from pathlib import Path

LOCALES_DIR = Path("locales")


def get_msgids(locale: str) -> set[str]:
    po_file = LOCALES_DIR / locale / "LC_MESSAGES" / "messages.po"
    po = polib.POFile(str(po_file))
    return {entry.msgid for entry in po if entry.msgid}


class TestI18nMessages:
    def test_all_english_keys_exist_in_japanese(self):
        en_ids = get_msgids("en")
        ja_ids = get_msgids("ja")
        missing = en_ids - ja_ids
        assert missing == set(), f"Missing in ja: {missing}"

    def test_all_japanese_keys_exist_in_english(self):
        en_ids = get_msgids("en")
        ja_ids = get_msgids("ja")
        missing = ja_ids - en_ids
        assert missing == set(), f"Missing in en: {missing}"

    def test_no_untranslated_entries(self):
        for locale in ["en", "ja"]:
            po_file = LOCALES_DIR / locale / "LC_MESSAGES" / "messages.po"
            po = polib.POFile(str(po_file))
            untranslated = [e.msgid for e in po.untranslated_entries()]
            assert untranslated == [], f"Untranslated in {locale}: {untranslated}"
```

## 12. Notes

| Item | Recommendation |
|------|------|
| Default locale | en (English) |
| Missing translations | Auto-check in CI with `pybabel compile` + tests |
| Long text | Verify API responses are correct in each language |
| Right-to-left (RTL) | Address when adding Arabic, etc. |
| .mo compilation | Always compile before deployment; add to CI |

---

## Summary

- Adopt **gettext + Babel** for standard Python i18n
- Manage translations as `.po` / `.mo` files in `locales/`
- Detect locale via `Accept-Language` header or `?lang=` query parameter using middleware
- Use `pybabel` for extraction, initialization, update, and compilation
- Format dates, numbers, and currencies with Babel locale-aware formatters
- Test translation completeness in CI
